## Phase 3（ゲームの汎用化 ＋ 会話形式）

**20 と 22 は企画方針の転換を含む**（20＝「FC版FE専用」の撤廃、22＝AIからの質問の解禁）。撤廃するのはその2つで、**攻略アドバイス禁止・大量注入禁止・RAG禁止・認証/公開の禁止は維持**する。FC版FE は引き続き第一の題材。

| No | チケット | 概要 | 主依存 |
|----|----------|------|--------|
| 20 | [ゲームを差し替え可能にする](./20-game-agnostic-core.md) | `knowledge/<slug>/` ＋ `game.json`・`playthroughs.game`・ラベルの差し替え | 05,06,07,09,12 |
| 21 | [`/knowledge` をゲームごとに差せるように](./21-knowledge-builder-per-game.md) | 抽出スキーマ・整形をゲーム定義から差す | 16,20 |
| 22 | [会話形式（質問ターンと返事待ち）](./22-conversational-turns.md) | `isIdle` → `turnKind`（4種）・関門4・タイムアウトで切り上げ | 04,07,10,13,15,18,19 |
| 23 | [ゲーム登録（primer 下書き生成）](./23-game-registration.md) | `/knowledge` からゲームを登録し `game.json`＋`primer.md` を書き出す | 16,20,21 |

前提として、**プロンプトの2層分離は済んでいる**（`src/lib/prompt.ts` ＝ 戦友の振る舞い／`knowledge/<game>/primer.md` ＝ そのゲームの前提）。20 に着手する前にここを混ぜ直さないこと。

### 実装順と依存（Phase 3）

```
20 ──┬─► 21 ──► 23        20 は 21/23 の土台（game.json・knowledge/<slug>/）
     │                     21 は 23 より先（game.json のスキーマを確定してから書き出す側を作る）
     └─► 22（独立）        22 は /knowledge に触らない。20 とも独立に成立する
```

**硬い制約は2つだけ**：**20 が最初**（21・23 の土台）、**21 は 23 より先**（`knowledgeBuilder` を含む `game.json` のスキーマが固まってから、それを書き出す 23 を作る）。22 はどこに置いてもよい。

推奨は **20 → 22 → 21 → 23**。理由は、22 が唯一「ループのタイミング」を触るチケットで（関門4・返事待ち）、19 で踏んだ地雷に最も近い＝**単独で A/B を取りたい**から。21・23 はどちらも `/knowledge` の作業なので、続けて片付けるほうが文脈が切れない。

1本終わるごとに `AS-BUILT.md` の該当箇所を更新し、次に進む前に止まって報告する。

## 進め方

- MVP の最短経路は **01→02→03→04→05→06→07→08→10**。09・11 は UX 仕上げ、12・13 は任意。
- Phase 2 は **14→15→16** の順（17 は独立・いつでも可）。14 だけで体感が変わるので、一度動かして話し方を確かめてから 15・16 を積む。
- Phase 3 は **20→21**。20 が終われば「ファイルを置くだけでゲームが増える」状態になる。21 は `/knowledge` を使いたいゲームが出てきてからでよい。
- 先読み実装をしない。`REQUIREMENTS.md` に無い機能は、まずチケットを起こす。
- 「やらないこと」の強い制約は **14・16・20 で一部が変わる**（ネタバレ禁止・参照サイト取得の禁止・複数ゲーム対応の禁止を順に撤廃。攻略アドバイス禁止は一貫して維持）。常に `REQUIREMENTS.md §5.2 / §12` を正とする。
