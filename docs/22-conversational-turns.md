# 22 会話形式にする（質問ターンと返事待ち）

## 概要

実況の一方通行より、**問いかけが混ざるほうが「一緒にやっている」実感が強い**（`REQUIREMENTS.md §2 / §5.2`）。ターンの種類に**質問**を足し、`isIdle`（真偽値）を **`turnKind`（4種）** に置き換える。

| turnKind | いつ | 何を喋る |
|----------|------|----------|
| `'narrate'` | 差分あり | 今この瞬間の実況（従来どおり） |
| `'chat'` | 差分なし | 昔話・雑談・励まし（旧 `isIdle: true`） |
| `'question'` | 確率 0.3 で抽選 | **答えなくても成立する軽い問いかけ** |
| `'giveup'` | 返事待ちのタイムアウト | **冒頭で軽く切り上げ、そのまま今の画面の話に続ける** |

**あいきょうの「厳格な1問1答」には戻さない。** 質問は投げっぱなしでよく、答えなくてもアプリは黙り込まない。

### ターン種別の決め方（`REQUIREMENTS.md §6.2`）

送信集約点（`send()`）で決める。**テンポ（乱数間隔）の仕組みには一切触らない。**

1. `userMessage` があれば**応答**を最優先（`turnKind` は無視・既存の挙動）。
2. 無ければ `QUESTION_TURN_PROBABILITY`（既定 **0.3**）で `'question'` を抽選。**ただし直前のAI発言が質問だったら抽選しない**（連続質問は尋問に聞こえる）。
3. 外れたら従来どおり差分しきい値で `'narrate'` / `'chat'` を出し分ける。

### 返事待ち（関門4）

質問を送ったら `awaitingAnswer` を立て、**質問の読み上げが終わってから** `QUESTION_ANSWER_TIMEOUT_MS`（既定 **90000**）を計る。読み上げ中から計り始めると、長い質問ほど待ち時間が短くなって不公平になる。

- 自動ループの関門に **④返事待ち中でない** を足す（①生成中でない → ②読み上げ中でない → ③時間 → ④返事待ち中でない）。
- **STT・手動トリガーは待ちを解除して割り込む**（既存どおり関門を通らない）。話しかけは `userMessage` として最優先で応答される＝これがそのまま「質問への回答」になる。手動トリガー（発話なし）も「先へ進みたい」の意思表示として待ちを解除する。
- タイムアウトしたら **`'giveup'` を1回だけ**送り、通常ループへ戻る。催促しない・蒸し返さない。**切り上げの一言だけの短い発言にはしない**（毎回だとくどい）。言い回しの繰り返しは、既存の「直近発言を渡して繰り返させない」仕組みに乗せる。
- UI に「返事待ち」を出す（黙っている理由が分かるように。**録画モードでは出さない**——視聴者に見せる画面ではない）。

関連: `REQUIREMENTS.md §2 / §5.2 / §6.2 / §7.1 / §7.3 / §12` ／ 依存: 04, 07, 08, 10, 13, 15, 18, 19（20 には依存しない）

## 設計メモ

- **抽選ロジックは純関数に切り出す**（`src/lib/turn.ts` の `pickTurnKind({ hasUserMessage, lastWasQuestion, diff, threshold, roll })`）。乱数は引数で受け取り、`node --experimental-strip-types` で確率も分岐も直接検証できるようにする（ブラウザを起動せずに済む唯一の部分）。
- **ターン指示は `/api/narrate` の4定数だけが持つ**（`NARRATE_TURN_TEXT` / `IDLE_TURN_TEXT` / `QUESTION_TURN_TEXT` / `GIVEUP_TURN_TEXT`）。システムプロンプトに同趣旨を書かない（15 の教訓）。`prompt.ts` に足すのは「質問してよい／答えを強要しない／質問を装って手順を誘導しない」という**振る舞いの許可と禁止だけ**で、「今このターンで質問しろ」は書かない。
- **`isIdle` は完全に置き換える**（両方を残さない。同じ意味の入力が2つあると、どちらが勝つかで必ず事故る）。

## 完了条件

- 実況・雑談・質問がランダムに混ざり、質問のあとは返事を待って黙る（ただし90秒まで）。
- 答えれば会話として拾い、答えなければ軽く切り上げて実況に戻る。**無音のまま止まり続けない。**
- 質問が2回続かない。手動トリガー・STT はいつでも割り込める。
- テンポ（30〜60秒の乱数間隔）の挙動は 19 から変わっていない。

## 注意

- **`QUESTION_ANSWER_TIMEOUT_MS` の待ちは「喋らない時間」＝動画の無音**。90秒は長い。実プレイで詰める前提の値。
- **質問を装った手順誘導は攻略アドバイス**（「シーダで話しかけてみたら？」）。`§12` の強い制約は変わらない。聞いてよいのは感想・思い出・気持ち・プレイヤー自身の選択。
- 関門を足すと**黙る条件が増える**。「壊れて黙っている」のか「返事を待っている」のか、UI と `console` で見分けられるようにしておく（19 で `[自発]` / `[変化]` を出したのと同じ理由）。
