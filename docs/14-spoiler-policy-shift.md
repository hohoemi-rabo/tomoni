# 14 仕様転換（ネタバレ解禁・発話量拡大）

## 概要

企画方針の転換。**「ネタバレしない」制約を撤廃**し、AIが自分から先の展開を語れるようにする。狙いは「誰かと一緒にやっている実感」で、沈黙がちで当たり障りのない相槌よりも、よく喋る語り部のほうが目的に適う。

**「攻略アドバイスをしない」は維持する。** 両者は別の軸である。

- ✅ 「あの赤い剣士、ナバールだよ。彼、実は仲間になるんだ」 → **事実**を語る（解禁）
- ❌ 「シーダで話しかけて」 → **手順**を指示する（引き続き禁止）

この線引きを一文にすると「**事実は語る、手順は言わない**」。戦友であって攻略先生ではない、という核は変わらない。

関連: `REQUIREMENTS.md §5.2, §7.3, §12` / 依存: 04, 06（06 の消費先は 07）

### 実装前に判明した取りこぼし（着手時に追記）

チケット起票時、以下4箇所を見落としていた。いずれもランタイムでLLMに効くため、これを直さないと「解禁したのに挙動が変わらない」で終わる。

- **`fe-primer.md`** はシステムプロンプト**先頭に固定注入**される（`prompt.ts:37`）。ここに禁止指示が残ると `prompt.ts` の変更が無効になる。
- **発話長の指示が4箇所に散在**（`prompt.ts` / `fe-primer.md` / `persona.tone` / `narrate/route.ts`）。しかも `persona.tone` は厳守リストの**末尾**に注入され（`prompt.ts:114`）、先に書いた「3〜4文」を打ち消す。→ **`prompt.ts` に一本化**する。
- **`persona` は作成時にDBへコピーされる**（`playthroughs.ts:57`）。`persona.ts` を直しても既存プレイスルーには届かない。→ SQL で更新する。
- **固有名ルールとネタバレ解禁が衝突する**。「ナバールは仲間になる」と語るには名前が要るが、`prompt.ts:50` は「画面に文字が無ければ固有名を断定するな」と縛る。→ 固有名ルールの適用範囲を「**いま画面に映っているユニットの特定**」に限定し、知識としての人物言及は妨げないと明示する。

## 完了条件

- AIが、到達していない章の展開や人物の運命を、聞かれずとも自分から語る。
- それでいて「この手が最適」「〜で話しかけろ」といった攻略指示は一切しない。
- 一度の発言が3〜4文程度あり、画面の小さな変化にも反応する。

## 注意

- **不可逆**：初見プレイは一度きり。この転換は開発者の明示的な意思決定に基づく。
- 最終成果物が YouTube 動画のため、**視聴者も同時にネタバレを受ける**。承知のうえ。
- `prompt.ts` は純関数。テストは `node --experimental-strip-types` で直接実行して確認する。
- `FRAME_DIFF_THRESHOLD` を下げるとGemini送信回数が増え、APIコストが増える。実機で体感とコストの折り合いを見る。
