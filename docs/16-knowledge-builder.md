# 16 章キャスト表の生成（参照URL入力UI）

## 概要

`knowledge/fe-fc/chapters/chapter-XX.md` を手で埋めるのが重い（現状 `chapter-01.md` は空テンプレのまま）。参照URL（最大3件）を入力し、取得 → 文字コード判定 → **表だけ抽出** → 章ごとの下書き生成 → **人間が目視確認して保存**、という一度きりの道具を用意する。

`/capture-test`・`/tts-test` と同じ「独立したハーネス」の位置づけ。**実況ループからは呼ばない。** 知識ファイル作りとセッションは寿命が違う。

関連: `REQUIREMENTS.md §8.4` / 依存: 05, 14

## 調査済みの前提（実測）

`https://izuito.net/game/fc-fireemblem/chart.htm` を検証済み。

- **1ファイルに全25章**の表が入っている。章タイトルはFC版と完全一致。
- 表の列は `No / 名前 / 職 / LV / HP / 力 / 技 / 武 / 早 / 運 / 守 / 移 / 魔 / 装備`。**`docs/05` のキャスト表テンプレとほぼ同じ項目**。
- 表の直後に `BOSS：ガザック／海賊 LV5 HP24 鋼の斧` の形式でボス情報がある。
- `No` の `S` 接頭辞（`S01 リフ` など・全23件）は「村や敵から加入する」印。**14 でネタバレを解禁したため、これは残してよい。**
- 表と散文は構造で分離できる。**散文は攻略手順（「地形効果を利用して…」「シーダで話しかけ仲間にしましょう」）なので全部捨てる。**
- Serenes Forest（英語・独立系）の第1〜2章の加入表と**照合して完全一致**。データの正しさは確認済み。
- `robots.txt` は全許可（`Disallow:` が空）。

## 完了条件

- URLを入れると25章分の下書きが出て、目視確認のうえ保存できる。
- 保存された `chapter-XX.md` に、**散文由来の攻略手順が1つも含まれていない**。
- Shift-JIS のサイトで文字化けしない。

## 注意

- **文字コードの罠（実測）**：`izuito.net` は Shift-JIS。素朴に取得して UTF-8 として読むと文字化けし、それをLLMに渡すと**もっともらしい嘘を生成する**（検証時、実際に「GBA版」「Map.1 マムスの矢道場」という存在しない情報が返ってきた）。charset 判定は必須。
- **自動でプロンプトへ流し込まない。** 生成物は必ず人間が確認してからファイルに落とす。汎用URL入力は「表と散文を機械的に見分ける」保証がない。
- 取得は一度だけ。サイトへ連続アクセスしない。取得した本文は**転載せず**、ローカルの知識ファイル生成にのみ使う。
- 25章分をディスクに置いてよい。`loadChapterCast` は `state.chapter` に対応する1枚しか読まないため、未到達章がプロンプトへ入ることはない。
- 固有名ルール（画面に文字が無ければ断定しない）は維持されるので、このキャスト表は**照合用**として働く。
