# 16 章キャスト表の生成（参照URL入力UI）

## 概要

`knowledge/fe-fc/chapters/chapter-XX.md` を手で埋めるのが重い。参照URL（最大3件）を入力し、取得 → 文字コード判定 → **表だけ抽出** → 章ごとの下書き生成 → **人間が目視確認して保存**、という一度きりの道具を用意する。

`/capture-test`・`/tts-test` と同じ「独立したハーネス」の位置づけ。**実況ループからは呼ばない。** 知識ファイル作りとセッションは寿命が違う。

関連: `REQUIREMENTS.md §8.4` / 依存: 05, 14

## 調査済みの前提（実測）

`https://izuito.net/game/fc-fireemblem/chart.htm` を検証済み。

- **1ファイルに全25章**の表が入っている。章タイトルはFC版と完全一致。
- 表の列は `No / 名前 / 職 / LV / HP / 力 / 技 / 武 / 早 / 運 / 守 / 移 / 魔 / 装備`。**`docs/05` のキャスト表テンプレとほぼ同じ項目**。
- 表の直後に `BOSS：ガザック／海賊 LV5 HP24 鋼の斧` の形式でボス情報がある。
- `No` の `S` 接頭辞（`S01 リフ` など・全23件）は「村や敵から加入する」印。**14 でネタバレを解禁したため、これは残してよい。**
- 表と散文は構造で分離できる。**散文は攻略手順（「地形効果を利用して…」「シーダで話しかけ仲間にしましょう」）なので全部捨てる。**
- Serenes Forest（英語・独立系）の第1〜2章の加入表と**照合して完全一致**。データの正しさは確認済み。
- `robots.txt` は全許可（`Disallow:` が空）。

### 実装時に分かったこと（次に触る人へ）

- **セルを改行にすると表が壊れる。** `</td>` を改行に置き換えると1行が縦に散らばり、LLM が列を対応づけられず **HP と持ち物が丸ごと落ちた**。セルは `|`、行は改行で区切る。
- **`responseSchema` の任意フィールドはモデルが黙って省略する。** `hp` / `items` を `required` にし `propertyOrdering` を明示するまで、指示文でいくら「埋めろ」と書いても埋まらなかった。読み取れないぶんは 0 / 空配列で返させ、整形側（`toUnit`）で落とす。
- **職業で敵味方を判断させない。** 敵から仲間になるユニット（ダロス＝海賊・カシム＝ハンター）が同じ自軍テーブルに載っているため、モデルが職業を見て敵に振り分けた。「ステータス表の行は全て allies」と明示して解決。
- **1章の失敗で全部を落とさない。** 25章を `Promise.all` で回すと1件の 503 で全滅する（実際に起きた）。失敗した章は `failed` として返し、UIで再取得を促す。累積が歯抜けになるため、黙って隠さない。
- **`withRetry` の既定は 503 に耐えられない。** 3回・500ms 起点＝合計1.5秒しか粘らず、Gemini の高負荷スパイク（数十秒）で章が欠ける（実運用で第19章が落ちた）。抽出は 5回・2秒起点（≒30秒粘る）にし、同時実行も 4→2 に下げた。25章で約2分かかるが、速さより完走を優先する。
- **失敗章より後ろは保存させない。** 自軍は累積なので、第19章が欠けるとチキが19〜25章の名簿から消える。失敗時のUIは「その章より前だけ保存し、残りは取り直す」と案内する。
- **`title` は番号ごと返ってくる**（`Map.1 マルスの旅立ち`）。`cleanTitle` で落とす。

## 既知の限界

- `izuito.net` には **ゴトー（第25章加入）が載っていない**。全章を通じた自軍は52人で、Serenes Forest の53人と1人ずれる。必要なら別のURLで補う（URL枠が3つあるのはこのため）。
- 実際の名簿はまだ保存していない。`chapters/` は空（空テンプレの `chapter-01.md` は削除した。中身が「（スクショから記入…）」のままプロンプトに注入されており、無いより悪かった）。**保存は開発者が `/knowledge` で目視確認して行う**。保存時に `chapters/` は自動で作られる。

## 注意

- **文字コードの罠（実測）**：`izuito.net` は Shift-JIS。素朴に取得して UTF-8 として読むと文字化けし、それをLLMに渡すと**もっともらしい嘘を生成する**（検証時、実際に「GBA版」「Map.1 マムスの矢道場」という存在しない情報が返ってきた）。charset 判定は必須。
- **自動でプロンプトへ流し込まない。** 生成物は必ず人間が確認してからファイルに落とす。汎用URL入力は「表と散文を機械的に見分ける」保証がない。
- 取得は一度だけ。サイトへ連続アクセスしない。取得した本文は**転載せず**、ローカルの知識ファイル生成にのみ使う。
- 25章分をディスクに置いてよい。`loadChapterCast` は `state.chapter` に対応する1枚しか読まないため、未到達章がプロンプトへ入ることはない。
- 固有名ルール（画面に文字が無ければ断定しない）は維持されるので、このキャスト表は**照合用**として働く。

## 21 で `game.json` 駆動に一般化した（2026-07-14）

抽出のゲーム固有部分（章見出しの正規表現・グループ＝自軍/敵・累積の有無・列・同定文）は、`knowledge/<game>/game.json` の `knowledgeBuilder` へ移した。

- `/api/knowledge/extract` と `/save` は `game` を受け取り、スキーマとプロンプトを定義から組む。**FE の設定は現行の挙動をそのまま書き下ろしてある**（同じURLで再生成し、既存 `chapter-01.md` と本文が一字一句同じであることを確認済み。差分は末尾の注記1行のみ）。
- `knowledgeBuilder` を持たないゲームでは `/knowledge` の章抽出を**使えない**（UIで無効化し、422 を返す）。章・ステージ構造を持たないゲームは primer 1枚で成立するので、これは劣化ではない。
- 純関数は `accumulateAllies` → `accumulateGroups`、`ChapterCast` は `{ allies, enemies }` → `{ groups: Record<string, CastUnit[]> }` に変わった。**上記の教訓（`required`/`propertyOrdering`・セル区切り・`Promise.all` で束ねない・リトライ5回/2秒起点・文字コード判定）はすべてそのまま生きている。**
