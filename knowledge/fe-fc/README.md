# knowledge/fe-fc — FC版FE 知識ファイル（攻略データではない）

AIの**感情・反応を正しくする前提**と、**今この章に誰がいるか**だけを持つ最小限の知識。2階建て構成（`REQUIREMENTS §8`）。

- `game.json` … ゲーム定義（§8.0・チケット20/21）。表示名・版・**進捗の呼び方**（`progressLabel` =「到達章」）・`lostLabel`、および `/knowledge` の**抽出設定**（`knowledgeBuilder`：章見出しの正規表現・グループ＝自軍/敵・累積の有無・列・同定文）。**AIの振る舞いも、ゲームの前提も書かない**（前者は `src/lib/prompt.ts`、後者は `primer.md`）。
- `primer.md` … 全章共通プライマー1枚。システムプロンプト先頭に固定。**このゲームの前提だけ**（同定・感情が動く場面・何が「手順」か・画面認識上の固有事情）。
- `chapters/chapter-XX.md` … 章ごとのキャスト表。`state.chapter` に対応する**1枚だけ**注入する（番号で引くだけの最小リトリーバル）。`chapter-01.md`, `chapter-02.md` … と**ASCII・ゼロ埋め2桁**で並べる。日本語の章タイトルは中の見出し（`# 第1章 …`）で保持。

ローダーは `src/lib/games.ts`（`loadGameDef` / `listGames`）と `src/lib/knowledge.ts`（`loadPrimer(game)` / `loadChapterCast(game, chapter)`）。どのディレクトリを読むかは `playthroughs.game` が決める。`state.chapter` が「第3章」なら `chapters/chapter-03.md` を読み、該当ファイルが無ければ未注入で続行する。

## ゲームを増やすには（チケット20/21）

**コードは触らない。** `knowledge/<slug>/`（slug は `[a-z0-9-]+`）を作り、`game.json` と `primer.md` を置くだけ。トップの新規作成フォームの選択肢に自動で現れる。章・ステージ構造を持たないゲームは `chapters/` を作らなくてよい（キャスト表は未注入で続行する）。

`/knowledge` で URL から章キャスト表を作りたいゲームだけ、`game.json` に `knowledgeBuilder` を足す（FE の書きぶりを参考に。章見出しの正規表現・グループ・累積の有無・列）。**無ければ `/knowledge` はそのゲームを無効表示にする**——「どんなゲームの表も読める1つの賢いスキーマ」は作らない方針（チケット21）。

## 章キャスト表の作り方 その1（参照URL → 名簿化・`/knowledge`・チケット16・§8.4）

`npm run dev` して `/knowledge` を開き、参照URL（最大3件）を入れて「取得して下書きを作る」。章ごとの下書きが出るので、**中身を目で見てから**保存する章を選ぶ。取り込むのは表の事実だけで、攻略手順の散文は捨てる。

- 参照先の**利用規約と `robots.txt` を確認**してから使う。取得は一度きりに留める（実況ループからは取得しない）。
- LLM は表を読み違えても静かに間違う。**目視確認を飛ばさない。**
- 「自軍」は第1章からの**累積**（その時点で画面にいる全員が要るため）。「敵」はその章のものだけ。
- 抽出に失敗した章があると、その章の仲間が抜けたまま以降に累積される。画面に警告が出たら、**失敗した章より前だけ保存し、失敗章とそれ以降は保存せずに再取得**する。
- 検証済みの参照元: `https://izuito.net/game/fc-fireemblem/chart.htm`（Shift-JIS・全25章）。ただし**ゴトー（第25章加入）が載っていない**ので、必要なら別のURLで補う。

## 章キャスト表の作り方 その2（スクショ → 名簿化・§8.4）


章を進めるたびに、その章の登場人物一覧の**スクリーンショットを撮って Claude Code に渡し**、テキスト名簿に起こす。指示には必ず次の3点を添える。

1. **§8.2 の体裁テンプレと同じ形式**で `chapters/chapter-XX.md` を作る（下記テンプレ）。
2. **スクショの表の情報だけ**をテキスト化する。**攻略・解説は書かない**（§5.2）。表に無いことを補わない。
3. ファイル名は **ASCII・ゼロ埋め番号**。日本語タイトルは中の見出しに。

### 体裁テンプレ（§8.2・統一する）

```markdown
# 第1章 マルスの旅立ち

## 自軍（仲間）
- マルス／ロード／Lv1 HP18／持ち物: レイピア
- ジェイガン／パラディン／Lv1 HP20／持ち物: 鉄の剣・銀の槍

## 敵
- ガザック／かいぞく／Lv5 HP24／持ち物: 鋼の斧（このマップのボス）
- ガルダへい（×11）／かいぞく／Lv1 HP18／持ち物: 斧
```

## 守ること

- **線引きは「事実は語る、手順は言わない」**（チケット14でネタバレは解禁・`REQUIREMENTS §5.2`）。人物の運命を書いてもよいが、**加入・攻略の手順**（誰で話しかける等）は書かない。そもそもキャスト表は画面と照合するための名簿であり、運命はAI自身の知識から語れる。無理に足さない。
- **マップ画像は含めない**（戦友の役割に不要。必要になれば新チケットを起こしてから検討）。
- **攻略サイトの本文・画像は転載しない**。誰がいるか＝事実を自分の言葉のテキスト名簿にする。
- 読み取れない値は空欄か省略。**表に無い項目を創作・補完しない**。
- 不確かな仕様は `⚠️要確認` を付け、**版の取り違えチェックは開発者が一次情報で行う**（FC版＝FE1・1990）。
